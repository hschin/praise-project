<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: End-to-end browser verification</name>
  <what-built>
    Complete lyrics pipeline: search form → ImportSongJob → Turbo Stream progress → ruby-annotated lyrics display → pinyin toggle → edit page → library search.
  </what-built>
  <how-to-verify>
    Run `bin/dev` (starts Rails server + Solid Queue worker together).

    **Test 1 — Claude-first recall (SONG-01, SONG-02, SONG-03, LIB-01):**
    1. Visit http://localhost:3000/songs
    2. In the search form, enter: `我願意` (or another well-known Chinese worship song)
    3. Click Import — you should be redirected to the song's show page in "processing" state
    4. Watch the progress steps update live: "Searching web → Fetching lyrics → Generating pinyin → Done"
    5. Confirm the final lyrics display shows Chinese characters with pinyin above each character
    6. Verify section headers are labeled in English (Verse 1, Chorus, Bridge, etc.)

    **Test 2 — Pinyin toggle (SONG-02/03):**
    1. On the completed song show page, click "Hide Pinyin"
    2. Confirm pinyin disappears (only Chinese characters remain)
    3. Click "Show Pinyin" — confirm pinyin reappears above characters

    **Test 3 — Failed import + manual paste (SONG-04):**
    1. Search for a clearly nonsense title: `xyznonexistentsong99999`
    2. Confirm the show page eventually shows the red error state
    3. Confirm a "Paste lyrics manually" form is visible in the error state
    4. Paste some Chinese text and click Import Pasted Lyrics
    5. Confirm the same show page transitions from failed → processing → done (no duplicate song created)

    **Test 4 — Manual paste always visible on index (SONG-04):**
    1. Visit http://localhost:3000/songs
    2. Confirm the "Paste lyrics manually instead" disclosure is visible without needing a failure first
    3. Click it — confirm the paste form expands

    **Test 5 — Library search and cards (LIB-02):**
    1. With at least 2 songs imported, type a partial title in the "Filter by title" search field
    2. Submit — confirm only matching songs appear
    3. Confirm each song card shows the first lyric line as preview text (not section count)

    **Test 6 — Edit song (LIB-03):**
    1. On any completed song's show page, click "Edit"
    2. Change a section label (e.g., "verse" → "pre-chorus") and modify one line of lyrics
    3. Click Save — confirm redirect back to show page with updated content

    **Test 7 — Hard refresh preserves state:**
    1. During an active import, hard-refresh the show page (Cmd+Shift+R)
    2. Confirm it shows the correct current state (not a blank page or stale "processing")

    Report any failures, visual glitches, or unexpected behavior.
  </how-to-verify>
