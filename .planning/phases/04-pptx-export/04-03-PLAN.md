<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Full PPTX export pipeline:
    - Export button on deck show page (decks/:id)
    - Background job (GeneratePptxJob) that calls Python script
    - Turbo Stream status updates: Generating... then Download link
    - Download endpoint serving the .pptx file
  </what-built>
  <how-to-verify>
    1. Open http://localhost:3000 and sign in.

    2. Navigate to any deck that has at least one song with Chinese lyrics (e.g. a deck built during Phase 3 testing).
       - If no such deck exists: create one, add a song from the library that has Chinese content.

    3. Click "Export PPTX" in the top-right of the deck show page.
       - Expected: button text changes to "Generating..." with a spinner immediately.
       - If the button does not change, check that Solid Queue worker (bin/jobs) is running.

    4. Wait up to 15 seconds for the Turbo Stream update.
       - Expected: button area updates to show "Download .pptx →" link (green) and "Ready" label.
       - If it shows an error message, note the error text.

    5. Click "Download .pptx →".
       - Expected: browser downloads a .pptx file named after the deck title.

    6. Open the downloaded .pptx in PowerPoint, Keynote, or LibreOffice Impress.
       - Check: Does it open without errors?
       - Check: Are Chinese characters visible (not empty rectangles or boxes)?
       - Check: Does pinyin appear as a smaller line above the Chinese text?
       - Check: Is the background color correct (matching the deck's theme)?
       - Check: Is the text color correct?

    7. Go back to the deck page and click "Export PPTX" again (re-export).
       - Expected: same Generating... → Download flow repeats successfully.

    Report any issues with specific steps.
  </how-to-verify>
