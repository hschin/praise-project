  <action>
    Create lib/pptx_generator/generate.py.

    Script reads JSON from stdin using sys.stdin.read() then json.loads(). Extracts deck data and output_path.

    SLIDE LAYOUT:
    - Presentation size: Pt(720) x Pt(405) (10in x 5.625in, standard widescreen, or use Inches(13.33) x Inches(7.5))
    - Use pptx.util.Inches, Pt, RGBColor from pptx.util and pptx.dml.color
    - Background: add_picture with a solid color image, OR set background fill to solid RGB color using slide.background.fill.solid() and slide.background.fill.fore_color.rgb = RGBColor(...)

    For each song, for each slide:
    1. Add a blank slide layout (slide_layouts[6] is blank)
    2. Set background to theme background_color (parse hex: RGBColor.from_string(color.lstrip('#')))
    3. Add a centered text box filling most of the slide (left/top margins ~10%, width/height ~80%)
    4. Split into pinyin block and Chinese block:
       - If slide has pinyin content: add pinyin as first paragraph(s) in the text box, font size = Chinese pt * 0.6
       - Add Chinese content as subsequent paragraph(s), font size from FONT_SIZE_MAP
    5. Set font for all runs: font.name = "Noto Sans SC" (python-pptx will embed the font name; the actual embedding requires the font to be installed on the generating machine OR use font.embed = True approach)

    FONT EMBEDDING approach: python-pptx does not automatically embed fonts by file path. The correct approach for CJK:
    - Set run.font.name = "Noto Sans SC" (or the exact name of the bundled font)
    - To force embedding, use the pptx XML directly: after building the presentation, walk the XML and set the <a:latin> and <a:ea> (East Asian) font elements. Set typeface="Noto Sans SC" on <a:ea> elements for CJK runs.
    - Alternatively: set font.name = "Noto Sans SC" and also set the East Asian font via the XML element: run._r.rPr.set(qn('a:ea'), ...) — see python-pptx docs on XML manipulation.

    SIMPLER APPROACH for CJK font rendering:
    - In python-pptx, for a run containing CJK text, set `run.font.name = "Noto Sans SC"` AND also manipulate the XML to set the eastAsianFont (ea) element to the same font name. This ensures Windows PowerPoint uses the correct font for CJK glyphs.
    - The font does NOT need to be embedded as a binary blob; it just needs to be named. On the target Windows machine with the font installed, it renders. For projectors without the font, we note in the README that font installation or pre-embedding is needed.
    - As a practical compromise for v1: set font.name to "Noto Sans SC", document in README that the font must be installed on the presenting machine, and note the CJK risk from STATE.md as still present.

    FONT_SIZE_MAP: {"small": 28, "medium": 36, "large": 48}
